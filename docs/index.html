Estructura de la Base de Datos

Tabla Clientes
  • id_cliente: int (identificador único)
  • nombre_cliente: str (nombre del cliente)
  • email: str (correo electrónico)
  • ciudad: str (lugar de residencia)
  • fecha_alta: date (fecha de registro)
Tabla Ventas
  • id_venta: int (identificador único)
  • fecha: date (fecha de la venta)
  • id_cliente: int (referencia a Clientes)
  • nombre_cliente: str (nombre del cliente)
  • email: str (correo electrónico)
  • medio_pago: str (método de pago utilizado)

Pseudocódigo y Diagrama de flujo

Pseudocódigo:

# Pseudocódigo de menu_kinder.py

1. Mostrar mensaje de bienvenida.
2. Mostrar las opciones del menú:
    - 1. Problema por investigar en Aurelion
    - 2. Dataset
    - 3. Pseudocódigo y Diagrama de flujo
    - 4. Sugerencias y mejoras aplicadas con Copilot
    - 0. Salir
3. Repetir mientras el usuario no elija "0":
    a. Pedir al usuario que escriba el número de su opción.
    b. Si la opción es "0":
        - Mostrar mensaje de despedida.
        - Terminar el programa.
    c. Si la opción es "1":
        - Mostrar mensaje sobre el problema a investigar.
    d. Si la opción es "2":
        - Mostrar el texto "DOCUMENTACION.md:".
        - Abrir el archivo DOCUMENTACION.md y mostrar su contenido.
        - Mostrar un punto final.
    e. Si la opción es "3":
        - Mostrar mensaje indicando que el pseudocódigo y diagrama de flujo están en los archivos del proyecto.
    f. Si la opción es "4":
        - Mostrar mensaje sobre sugerencias y mejoras aplicadas con Copilot.
    g. Si la opción no es válida:
        - Mostrar mensaje de error.
